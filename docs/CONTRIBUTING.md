# Contributing Guidelines

This file specifies guidelines on how to contribute to this repository. We welcome contributions from the community and are pleased that you are interested in helping us improve this project!

### 1. Reporting Bugs
Before creating bug reports, please check the issue list to see if the problem has already been reported. When you are creating a bug report, please include as many details as possible:
* **Use a clear and descriptive title.**
* **Describe the exact steps to reproduce the problem.**
* **Describe the behavior you observed** and point out how it differs from what you expected.
* **Include screenshots or logs** if relevant.

### 2. Suggesting Enhancements
If you have an idea to improve the project, please open an issue with the tag `enhancement`. Explain **why** this enhancement would be useful to other users and how it should work.

### 3. Code Contributions
We follow the standard "Fork & Pull" workflow.

## üõ†Ô∏è Pull Request Process

1.  **Fork the Repository:** Click the "Fork" button at the top right of this page to create your own copy of the repository.
2.  **Clone the Fork:** Clone your fork to your local machine.
    ```bash
    git clone https://github.com/riya-pesu/Crowd-Monitoring-and-Cluster-Detection-System/
    ```
3.  **Create a Branch:** Always work on a new branch, not `main`. Use a descriptive name.
    ```bash
    git checkout -b feature/your-feature-name
    # or
    git checkout -b fix/issue-you-are-solving
    ```
4.  **Make Changes:** Implement your features or fixes. Please adhere to the existing coding style of the project.
5.  **Test Your Changes:** Ensure your code works as expected and doesn't break existing functionality.
6.  **Commit Changes:** Write clear, concise commit messages.
    ```bash
    git commit -m "Add feature X to handle Y"
    ```
7.  **Push to GitHub:**
    ```bash
    git push origin feature/your-feature-name
    ```
8.  **Submit a Pull Request:** Go to the original repository on GitHub and click "New Pull Request". Select your branch and describe your changes.

## üìù Style Guidelines

* **Code Style:** Try to match the existing indentation and formatting of the files you are editing.
* **Commit Messages:**
    * Start with a verb (e.g., "Add", "Fix", "Update").
    * Keep the first line under 50 characters if possible.
    * Reference issue numbers if applicable (e.g., "Fixes #42").
